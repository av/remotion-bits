# BackgroundTransition

The `BackgroundTransition` component enables smooth transitions between CSS gradients, intelligently interpolating colors, angles, and gradient types.

## Installation

```bash
npm install remotion-bits
```

Or via jsrepo:

```bash
npx jsrepo add background-transition
```

## Basic Usage

```tsx
import { BackgroundTransition } from 'remotion-bits';

export const MyComponent = () => (
  <BackgroundTransition
    gradient={[
      "linear-gradient(0deg, #667eea 0%, #764ba2 100%)",
      "linear-gradient(180deg, #f093fb 0%, #f5576c 100%)"
    ]}
    duration={90}
  >
    <YourContent />
  </BackgroundTransition>
);
```

## Props

### `gradient`

**Type:** `string[]` (required)

Array of CSS gradient strings to transition between. Supports:
- `linear-gradient()`
- `radial-gradient()`
- `conic-gradient()`

Each gradient can have any number of color stops.

```tsx
gradient={[
  "linear-gradient(45deg, red, blue)",
  "radial-gradient(circle, yellow, green)"
]}
```

### `duration`

**Type:** `number` (required)

Total duration of the transition in frames.

### `easing`

**Type:** `string | ((t: number) => number)`
**Default:** `"linear"`

Easing function for the transition. Can be:
- A preset string (see [Easing](/docs/utilities/easing))
- A custom function `(t: number) => number` where t is 0-1

```tsx
easing="easeInOutCubic"
// or
easing={(t) => t * t * t}
```

### `children`

Content to render on top of the gradient background.

### `style`

Additional CSS styles to apply to the container.

### `className`

CSS class name for the container element.

## How It Works

The component intelligently handles gradient transitions:

1. **Color Interpolation**: Colors are converted to OKLch color space for perceptually uniform transitions
2. **Angle Interpolation**: Gradient angles take the shortest rotational path
3. **Type Morphing**: Smoothly transitions between linear, radial, and conic gradients
4. **Stop Alignment**: Automatically aligns color stops between gradients with different stop counts

## Advanced Usage

### Looping Gradients

Create cycling gradient effects by providing multiple gradients:

```tsx
<BackgroundTransition
  gradient={[
    "linear-gradient(0deg, #667eea, #764ba2)",
    "linear-gradient(120deg, #f093fb, #f5576c)",
    "linear-gradient(240deg, #4facfe, #00f2fe)",
    "linear-gradient(360deg, #667eea, #764ba2)" // Same as first
  ]}
  duration={180}
  easing="easeInOut"
>
  <Content />
</BackgroundTransition>
```

### Complex Multi-Stop Gradients

Mix different numbers of color stops:

```tsx
<BackgroundTransition
  gradient={[
    // 2 stops
    "linear-gradient(45deg, #667eea 0%, #764ba2 100%)",
    // 5 stops
    "linear-gradient(135deg, #667eea 0%, #764ba2 25%, #f093fb 50%, #4facfe 75%, #00f2fe 100%)"
  ]}
  duration={120}
>
  <Content />
</BackgroundTransition>
```

The component automatically interpolates intermediate stops.

### Angle Shortcuts

Angles take the shortest path. For example, 350째 to 10째 rotates 20째 clockwise rather than 340째 counter-clockwise.

```tsx
<BackgroundTransition
  gradient={[
    "linear-gradient(350deg, #fa709a, #fee140)",
    "linear-gradient(10deg, #30cfd0, #330867)"
  ]}
  duration={90}
>
  <Content />
</BackgroundTransition>
```

## Tips

1. **Color Formats**: Use any CSS color format (hex, rgb, hsl, oklch). They're automatically converted for smooth interpolation.

2. **Performance**: Gradient animations are GPU-accelerated via CSS, making them very performant.

3. **Composition**: The component uses `AbsoluteFill` from Remotion, so children can be positioned absolutely.

4. **Color Stops**: When transitioning between gradients with different numbers of stops, intermediate stops are automatically generated.
