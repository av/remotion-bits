# Easing Functions

Remotion Bits includes a comprehensive set of easing functions for creating natural, polished animations. All easing functions are based on Robert Penner's easing equations.

## Import

```tsx
import { EASING } from 'remotion-bits';
// or import individual functions
import { easeOutQuad, easeInOutCubic } from 'remotion-bits';
```

## Usage

### With Interpolate

```tsx
import { interpolate, easeOutQuad } from 'remotion-bits';

const opacity = interpolate(frame, [0, 30], [0, 1], {
  easing: easeOutQuad
});
```

### With String Names

```tsx
import { interpolate } from 'remotion-bits';

const opacity = interpolate(frame, [0, 30], [0, 1], {
  easing: 'easeOutQuad'
});
```

### With Components

```tsx
<TextTransition
  transition={{
    opacity: [0, 1],
    easing: 'easeOutQuad'
  }}
>
  Text
</TextTransition>
```

## Available Easing Functions

### Linear

No easing, constant speed.

```tsx
easing: 'linear'
```

**Use case:** Mechanical movements, technical animations

### Quadratic (Quad)

Gradual acceleration/deceleration.

```tsx
easing: 'easeInQuad'    // Starts slow
easing: 'easeOutQuad'   // Ends slow
easing: 'easeInOutQuad' // Starts and ends slow
```

**Use case:** Gentle animations, subtle transitions

### Cubic

More pronounced acceleration/deceleration than Quad.

```tsx
easing: 'easeInCubic'
easing: 'easeOutCubic'
easing: 'easeInOutCubic'
```

**Use case:** Standard UI animations, most common choice

### Quartic (Quart)

Strong acceleration/deceleration.

```tsx
easing: 'easeInQuart'
easing: 'easeOutQuart'
easing: 'easeInOutQuart'
```

**Use case:** Dramatic entrances/exits

### Quintic (Quint)

Very strong acceleration/deceleration.

```tsx
easing: 'easeInQuint'
easing: 'easeOutQuint'
easing: 'easeInOutQuint'
```

**Use case:** Impactful animations, hero elements

### Exponential (Expo)

Explosive acceleration/deceleration.

```tsx
easing: 'easeInExpo'
easing: 'easeOutExpo'
easing: 'easeInOutExpo'
```

**Use case:** Explosive effects, zoom animations

### Circular (Circ)

Circular arc acceleration/deceleration.

```tsx
easing: 'easeInCirc'
easing: 'easeOutCirc'
easing: 'easeInOutCirc'
```

**Use case:** Smooth, flowing animations

### Sine

Sinusoidal acceleration/deceleration.

```tsx
easing: 'easeInSine'
easing: 'easeOutSine'
easing: 'easeInOutSine'
```

**Use case:** Very gentle, natural animations

### Back

Overshoots then settles (anticipation effect).

```tsx
easing: 'easeInBack'    // Pulls back before moving
easing: 'easeOutBack'   // Overshoots then settles
easing: 'easeInOutBack' // Both effects
```

**Use case:** Playful animations, button interactions

### Elastic

Spring-like bouncy effect.

```tsx
easing: 'easeInElastic'
easing: 'easeOutElastic'
easing: 'easeInOutElastic'
```

**Use case:** Attention-grabbing animations, notifications

### Bounce

Bouncing ball effect.

```tsx
easing: 'easeInBounce'
easing: 'easeOutBounce'
easing: 'easeInOutBounce'
```

**Use case:** Playful landing animations, fun interactions

## Common Patterns

### Default Recommendations

```tsx
// Text appearing
easing: 'easeOutQuad'

// Elements entering screen
easing: 'easeOutCubic'

// Elements exiting screen
easing: 'easeInCubic'

// Smooth bidirectional
easing: 'easeInOutCubic'

// Attention-grabbing
easing: 'easeOutBack'

// Playful
easing: 'easeOutElastic'
```

### Combining Easing Functions

```tsx
// Fade in with ease out, move with ease in out
<MotionTransition
  transition={{
    opacity: {
      values: [0, 1],
      easing: 'easeOut'
    },
    y: {
      values: [100, 0],
      easing: 'easeInOutCubic'
    }
  }}
>
  <div>Content</div>
</MotionTransition>
```

## Custom Easing Functions

Create your own easing function:

```tsx
// Custom cubic bezier
const customEasing = (t: number) => {
  return t < 0.5
    ? 4 * t * t * t
    : 1 - Math.pow(-2 * t + 2, 3) / 2;
};

<TextTransition
  transition={{
    opacity: [0, 1],
    easing: customEasing
  }}
>
  Text
</TextTransition>
```

### Easing Function Signature

```tsx
type EasingFunction = (t: number) => number;
// t: progress from 0 to 1
// returns: eased value from 0 to 1
```

### Example Custom Easing

```tsx
// Custom bounce
const customBounce = (t: number) => {
  const n1 = 7.5625;
  const d1 = 2.75;

  if (t < 1 / d1) {
    return n1 * t * t;
  } else if (t < 2 / d1) {
    return n1 * (t -= 1.5 / d1) * t + 0.75;
  } else if (t < 2.5 / d1) {
    return n1 * (t -= 2.25 / d1) * t + 0.9375;
  } else {
    return n1 * (t -= 2.625 / d1) * t + 0.984375;
  }
};

// Custom elastic
const customElastic = (t: number) => {
  const c4 = (2 * Math.PI) / 3;
  return t === 0 ? 0
    : t === 1 ? 1
    : -Math.pow(2, 10 * t - 10) * Math.sin((t * 10 - 10.75) * c4);
};
```

## Performance Considerations

1. **Precomputed**: All built-in easing functions are optimized
2. **Pure functions**: No side effects, can be cached
3. **No dependencies**: Easing functions have no external dependencies

## EASING Object

Access all easing functions via the `EASING` constant:

```tsx
import { EASING } from 'remotion-bits';

const easingFunction = EASING.easeOutCubic;
const value = easingFunction(0.5); // 0.875
```

Available on the EASING object:
- `linear`
- `easeInQuad`, `easeOutQuad`, `easeInOutQuad`
- `easeInCubic`, `easeOutCubic`, `easeInOutCubic`
- `easeInQuart`, `easeOutQuart`, `easeInOutQuart`
- `easeInQuint`, `easeOutQuint`, `easeInOutQuint`
- `easeInExpo`, `easeOutExpo`, `easeInOutExpo`
- `easeInCirc`, `easeOutCirc`, `easeInOutCirc`
- `easeInSine`, `easeOutSine`, `easeInOutSine`
- `easeInBack`, `easeOutBack`, `easeInOutBack`
- `easeInElastic`, `easeOutElastic`, `easeInOutElastic`
- `easeInBounce`, `easeOutBounce`, `easeInOutBounce`

## Tips

1. **Start with easeOutCubic**: It's the most commonly used and works for most cases

2. **In vs Out vs InOut**:
   - Use `easeOut` for elements entering
   - Use `easeIn` for elements exiting
   - Use `easeInOut` for continuous animations

3. **Matching duration**: Stronger easing functions (Quint, Expo) work better with shorter durations

4. **Consistency**: Use similar easing functions across your video for cohesive feel

5. **Test in context**: Preview animations at actual speed to ensure they feel right
